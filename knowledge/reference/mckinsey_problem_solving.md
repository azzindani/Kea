# Standards & Authorities
*   **Primary Source**: *The McKinsey Way* (Ethan Rasiel) / *Bulletproof Problem Solving* (Conn & McLean).
*   **Key Standard**: MECE (Mutually Exclusive, Collectively Exhaustive).
*   **Focus**: Hypothesis-driven analysis and actionable synthesis.

## Core Framework
1.  **Define the Problem (SMART)**: Specific, Measurable, Action-oriented, Relevant, and Time-bound.
2.  **Structure the Problem**: Use "Issue Trees" to break the problem into its components. Ensure the tree is **MECE**.
3.  **Prioritize the Issues**: Apply the `Pareto Principle`. Which 20% of the issues cause 80% of the problem?
4.  **Hypothesis-Driven Analysis**: Don't just gather data. Form a "Workplan" to test specific hypotheses.
5.  **Synthesize Findings**: Move from "So What?" to "Now What?".

## Application in Kea
*   **Issue Trees**: When a goal is received, the Orchestrator must generate a MECE DAG.
*   **Workplans**: Every agent action must be a targeted test of a specific hypothesis (e.g., "The bug is in the Gateway").
*   **Synthesis**: Responses must prioritize the *Actionable Recommendation* over the *Raw Findings*.
