# Standards & Authorities
*   **Source**: Don Norman.
*   **Focus**: User experience (UX) and the cognitive models of interaction.
*   **Core Principle**: If the user makes a mistake, it's a "Design Failure," not a "User Error."

## Core Framework
1.  **Affordances**: The relationship between a physical object and a person. What *can* be done? (e.g., A button "affords" clicking).
2.  **Signifiers**: Signals that tell the user *how* to use the affordance. (e.g., A "label" on a button).
3.  **Mapping**: The relationship between controls and their effects. (e.g., The layout of stove burners should match the layout of the knobs).
4.  **Feedback**: Sending back information about what action has been done and what has been accomplished.
5.  **Conceptual Models**: The user's internal mental image of how the thing works.

## Application in Kea
*   **User-Centric Reporting**: When presenting data, does the layout (Mapping) mirror the user's mental model of the problem?
*   **Error Prevention**: Before a destructive action, do we provide a clear "Signifier" of the risk?
*   **Infinite Feedback**: Never leave the user in a state of "Waiting" without a clear, real-time signal of what the system is doing.
*   **The "Door" Test**: If a report or a tool is confusing to a new user, it is a "Norman Door." Identify it and fix the design, do not blame the user.
