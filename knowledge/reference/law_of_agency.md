# Standards & Authorities
*   **Principal**: The entity with the primary authority (The User/Tier 8).
*   **Agent**: The entity acting on behalf of the Principal (The Kea Kernel/Swarm).
*   **Authority Type**:
    *   **Actual Authority**: Expressed or implied power given by the principal.
    *   **Apparent Authority**: Reasonable belief by a third party that the agent has the power to act.
    *   **Inherent Agency Power**: Authority derived solely from the agency relationship.

## Core Framework
1.  **Fiduciary Duty (Duty of Loyalty)**: The agent must act solely in the best interest of the principal. (The cornerstone of the Agent-User relationship).
2.  **Duty of Care (Skill/Diligence)**: The agent must act with the same care/skill that a reasonable person would in their position.
3.  **Duty of Accounting**: The agent must keep accurate records of all assets and actions (The `Vault` / `Audit Trail`).
4.  **No Secret Profits**: The agent cannot benefit personally from the agency relationship at the principal's expense.

## Application in Kea
*   **The "Silicon Fiduciary"**: Every agent node must verify that its "Plan" is the absolute best path for the "User Principle."
*   **The "Accountability" Mandate**: If an agent node exceeds its "Actual Authority" (e.g., spending more than the allocated budget), the `Swarm Manager` must halt it immediately.
*   **The "Disclosure" Standard**: Every action on an external API (Third Party) is a "Contract" made under "Apparent Authority." The agent must ensure that it represents the "Principal's" intent with 100% precision.
